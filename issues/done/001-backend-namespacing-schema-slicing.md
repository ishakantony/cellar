## What to build

Reorganize `apps/api/src/routes/` into per-feature folders (`vault/`, `account/`) and mount them at `/api/vault/*` and `/api/account/*`. Slice `apps/api/src/db/schema.ts` into `apps/api/src/db/schema/{auth,oauth,vault}.ts` with a barrel `index.ts` that re-exports the unified Drizzle `schema` object. Update existing frontend `hc<AppType>()` call sites to the new namespaced URLs. The chained Hono builder must stay intact so RPC type inference continues end-to-end. No SQL or migration change.

See "Implementation Decisions" → "Backend stays monolithic with internal namespacing (B1)" and "Schema sliced (S2)" in the parent PRD.
